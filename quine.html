Quine McCluskey Techniques

The Quine McCluskey Method is also referred to as the Tabular Method. It was created by Willard V. Quine in 1952 and extended by Edward J. McCluskey in 1956. It implements searching rather than mapping and collects terms by looking for entries that differ only in a single bit. It is more useful than K-mapping when the number of inputs is greater than 6.

Willard V. Quine

Decimal-Binary Converter

Quine McCluskey Techniques are dependent on the binary representation of min-terms. Use this tool for your conversions.

Key Terms

Implicant

A group of ones.

Prime Implicant

The largest possible groups of ones.

Essential Prime Implicant

A prime implicant having at least one min-term that can’t be combined in any other way.

Bit Hierarchy

In the function f(A, B, C, D), the most significant bit (MSB) is A and D is the least significant bit (LSB).

Let's Take a Look at an Example!

Y(A,B,C,D) = Σm(0, 2, 6, 8, 9, 10, 11, 15)

Before we get started, we must understand what the question is saying. A, B, C and D are the variables. 0, 2, 6, etc are the min-terms.

Step 1: Binary Equivalents

The first step is to identify the binary equivalents of each min-term. We can do this using the 'Decimal-Binary Converter' above. The number of bits used to represent the min-terms is determined by the number of variables present. Since there are four variables, we can have a maximum of 2^4 min terms (i.e. we can represent min-terms from 0-15)

Step 2: Group Min-Terms

The second step is to create a table to group the min-terms based on the number of ones present in each binary representation. The Group number corresponds to the number of 1's present in the Binary Representation. 'M' refers to the min-term.

Step 3: Matching Pairs

The third step is to create a table of matching pairs using the previous table. Each pair will have a member from nth and (n+1)th groups. The members of each pair can ONLY differ by one bit. Step 4: Second Matching Pairs

Since the previous table can be minimized further, we create another Matching Pair table. The same rules apply. Here are some examples. The video below shows how all pairings were achieved. This table will produce the Prime Implicants. 1 represents the variable and 0 represents the complement of the variable. The last slide is a representation of what not to do. As you can see, (M2-M6) and (M6-M14) differ by 2 bits.

Step 5: Prime Implicant Table

The final step is to create the Prime Implicant Table. This table will determine the Essential Prime Implicants which will make up the minimized function. Place X’s in the columns where the min-terms are represented. We then highlight the X’s in the columns which only contain 1 X. Afterwards, we highlight these rows to determine the Essential Prime Implicants to create the minimized expression.
